| `tx_switch.kicad_sch` | ❌ Crashes | SIGSEGV |
| Root (`turboquant_mux_lna_v5.kicad_pro`) | ❌ Fails | "Failed to load schematic" |

**Root cause identified:**
- `power.kicad_sch` has **cached `lib_symbols`** definitions for all its components → works standalone
- `analog`, `tx_switch`, `digital` sheets have **empty/minimal `lib_symbols`** → rely on external KiCad standard libraries
- When KiCad tries to resolve symbols from external libraries (`Analog_Switch:DG408`, `Amplifier_Operational:OPA1641`, etc.), it crashes or fails

Standard KiCad libraries ARE installed at `/usr/share/kicad/symbols/` (verified: `74xx.kicad_sym`, `Amplifier_Current.kicad_sym`, etc. exist). The crash is likely a KiCad 9.0.7 bug with hierarchical schematic resolution, or the project library search path is not configured.

**Project file issue:**
- `turboquant_mux_lna_v5.kicad_pro` has `"pinned_symbol_libs": []` and `"legacy_lib_list": []`
- No library search paths configured → KiCad can't resolve symbols
