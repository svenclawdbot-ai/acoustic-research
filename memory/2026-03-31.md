## 🎯 Major Achievement: FPGA Controller Architecture Decision

After reviewing both PCB designs:
- **Red Pitaya Mux Board**: Has CD4051B analog muxes, requires parallel 8-channel ADC interface
- **TurboQuant Mux LNA**: Uses 74HC595 shift register for sequential element selection

**Decision**: Use **TurboQuant board** with **sequential scanning approach**.

Rationale:
- Switching time ~10μs (8 bits @ 1MHz SPI) is fast enough for ultrasound scanning
- Reduces FPGA complexity from 8 parallel ADC inputs to 3 digital control lines
- Uses Red Pitaya's built-in fast ADCs (IN1/IN2 at 125 MSa/s) for RX
- Much simpler Verilog implementation

## 🔧 TurboQuant Interface Specification

### Control Signals (E1 Connector)
| Signal | Pin | Description |
|--------|-----|-------------|
| SER | DIO0_P (G18) | Serial data to 74HC595 |
| SRCLK | DIO0_N (G17) | Shift register clock |
| RCLK | DIO1_P (H17) | Latch clock |

### Timing
- SPI Clock: 1 MHz (derived from 125 MHz system clock)
- Shift out: 8 bits × 1 μs = 8 μs
- Latch: < 1 μs
- **Total switch time: ~10 μs**
- **Max scan rate: 100,000 elements/second**

### Data Flow
```
Python Host → [element_select: 0-7] → FPGA Controller → 74HC595 → GATE0-7 → TurboQuant
                                         ↓
Red Pitaya ADC IN1/IN2 ←──────────────── RX Signal
```

## 💡 Key Technical Insight

The parallel beamformer (8-channel simultaneous) would require:
- 8 parallel ADCs on the mux board (not present)
- Complex delay-line synchronization in FPGA
- Higher pin count on E1 connector

Sequential approach:
- Leverages existing 74HC595 on TurboQuant board
- Simple 3-wire interface
- Software-controlled scanning pattern
- Can still achieve beamforming via software delay-and-sum post-processing
